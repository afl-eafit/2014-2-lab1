Programming lab 1: Simulating a non-deterministic finite automaton

Submission deadline: Friday, September 5, 2014 (11:59 pm)

Introduction

The aim of this programming assigment is to write a Haskell program that given a non-deterministic finite automaton nfa and a word w, determines if w ∈ L(nfa).

Assignment (80%)

  • Download from the GitHub repository cb0081-2014-2-lab1 the modules NFA.hs and Example.hs, where you will find an implementation and an example of a non-deterministic finite automaton, respectively.
  • (70%) Using the module NFA, define a function 
          simulatingNFA :: NFA states symbols -> [symbols] -> Bool
    where simulatingNFA nfa w determines if w ∈ L(nfa).
  • (10%) Annotate your source code in a way that explains your solution for the assignment. Improve it with HLint.

Coding style (10%)

Before submitting your programming lab verify that your code:

  • Has type signatures for everything (even where clauses and let expressions).
  • Has good (English) comments.
  • Has no overly complicated function definitions.
  • Does not contain any repetitive code.
  • Has a consistent layout.
  • Does not have long lines (at most 80 columns).
  • Has no junk (unused code, commented code, unnecessary code).
  • Does not have trailing whitespace (delete all unnecessary spaces at the end of a line, or empty lines at the end of a file). See fixing trailing whitespace issues with Emacs.

Submission (10%)

Submission is electronic. Send an email to asr(at)eafit(dot)edu(dot)co and dmonto39(at)eafit(dot)edu(dot)co and attach a compressed file with your solution. Your submission has to include the following:

  • SimulationNFA.hs, a Haskell module file with your solution for the assigment.
  • README.txt, a file containing at least the following information.
    • The name of your program.
    • Your name(s).
    • A general description of your program.
    • Information on how to use your program.

Required GHC version

You are to use GHC 7.8.3. You can get it installing the Haskell Platform 2014.2.0.0 (recommended) or you can install it from the Glasgow Haskell Compiler.

Additional information

  • The programming lab may either be solved on your own, or jointly with one other student taking the course.
  • You can add class constraints to the type signature of the simulatingNFA function if you need them.
